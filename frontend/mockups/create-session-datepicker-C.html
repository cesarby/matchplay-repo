Detalles de implementación

  • • Librería: react-day-picker (12kb gzip, accesible, headless con CSS propio)
  • • Trigger compacto cerrado, popover ancla bajo el campo con el calendario
  • • Días pasados deshabilitados (tachados, no clickables); días vacíos del mes muted
  • • Hoy en amarillo border + tinte; selección en rojo sólido con sombra
  • • Strip de horas 18→22 (las más típicas de partida), con opción de input libre debajo (no en mockup)
  • • Puntito verde en días con partidas conocidas — datos del backend `/sessions?scheduledFrom=...` (opcional, gancho futuro)
  • • Coste real: 3-4 horas (install, theme, integración con react-hook-form, tests a11y)
